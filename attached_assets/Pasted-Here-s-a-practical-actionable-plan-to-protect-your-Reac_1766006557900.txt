## üõ° 3. **React-specific security best practices**

Even after dependency updates, secure coding practices minimize risk from common attacks like XSS, broken auth, etc.:

**XSS / content injection**

* Avoid `dangerouslySetInnerHTML` unless absolutely necessary. If you must, *sanitize* user content with a library like **DOMPurify** before rendering. ([DEV Community][6])

**Avoid exposing secrets**

* Don‚Äôt embed API keys or credentials in the frontend bundle; treat all frontend code as public. ([nareshit.com][7])

**Authorization checks**

* Perform sensitive authorization on the *server* (backend API), not just client-side guards. ([DEV Community][8])

**Regular scanning**

* Integrate security scanners into CI/CD to catch issues early (linting, audit tools, dependency checks). ([Cyber Sierra][9])
